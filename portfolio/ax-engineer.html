Case 01 · Development

PR 기반 AI 릴리즈 노트 및 Smoke Test 자동화

문제기획자 부재로 개발자가 릴리즈 직전 코드를 다시 읽으며 변경 문서를 수기 작성성과명시적 변경 누락률 1% 미만

PR을 단순 코드 병합 단위가 아니라 변경 설명과 실행 검증의 시작점으로 사용했습니다.

1. 변경 분석
PR 생성 시 Agent가 코드 변경을 분석해 사용자 영향, 내부 변경, 주의사항을 포함한 릴리즈 노트 초안을 생성합니다.
2. 사람 리뷰
개발자가 초안을 검토하고 의도와 다른 설명을 수정합니다. AI 결과가 곧바로 공식 문서가 되지 않게 했습니다.
3. 실행 Smoke
핵심 실행 경로를 Smoke Test로 검증해 문서상 변경과 실행 가능한 결과가 함께 승인되도록 구성했습니다.
4. 릴리즈 병합
릴리즈 시점에는 승인된 변경 설명만 모아 최종 문서로 병합했습니다.

통제 포인트

생성은 Agent가 담당하지만 승인 책임은 개발자에게 두고, 실행 Smoke 결과를 함께 남겨 설명과 실제 동작의 불일치를 줄였습니다.

결과: 릴리즈 직전 수기 재조사 부담을 줄이고 명시적 변경 누락률을 1% 미만으로 낮춤

Case 02 · QA

스크린샷 기반 TC·Jira·Git 연계 자동화

문제수기 테스트 이후 TC 갱신, 증적 정리, Jira 작성, 개발 연계에 반복 시간이 발생성과QA 수동 검수·행정시간 30% 이상 단축

테스트 행위 자체를 무리하게 자동화하지 않고, 결과를 옮기고 설명하고 연결하는 주변 행정업무부터 제거했습니다.

1. 기존 행동 유지
QA는 기존처럼 테스트하고 실패 화면의 스크린샷을 저장합니다.
2. 자동 매핑
파일명과 테스트 규칙을 이용해 스크린샷을 TC와 연결하고 재현 단계와 증적 후보를 구성합니다.
3. 이슈·코드 연계
Jira 티켓 생성, 증적 첨부, 관련 Git 작업 연결을 자동 수행합니다.
4. 예외 검토
판정이 모호하거나 중복 가능성이 있는 건만 QA가 최종 검토합니다.

통제 포인트

모든 쓰기 작업을 직렬화하고, 자동 생성 결과는 최종 상신 전에 QA가 확인하도록 분리했습니다.

결과: 현업 테스트 방식의 변화 없이 검수 및 행정 처리시간을 30% 이상 단축

Case 03 · HR

비정형 전자결재 기반 휴가 관리 자동화

문제ERP 미지원 휴가를 Excel로 관리하고, 전자결재 문서는 표준 양식 없이 자유서술성과휴가 문서 관리시간 99% 감소

규칙 기반 처리가 어려울 정도로 제각각 작성된 문서를 Agent가 구조화하되, 원문과 수정 이력을 항상 남겼습니다.

1. 문서 수집
승인된 전자결재 문서를 웹에서 수집하고 기안자와 원문 링크를 식별합니다.
2. 비정형 해석
Agent가 휴가 유형, 사용일, 시간 단위, 발생 근거를 구조화합니다.
3. Excel 갱신
Excel COM 자동화로 기존 휴가 대장을 갱신해 담당자의 익숙한 업무 화면을 유지합니다.
4. 근거 보존
내부 DB에 원문 링크, 추출 결과, 수정 이력과 처리 시각을 저장합니다.

통제 포인트

문서 해석 결과가 충돌하거나 날짜·휴가 유형이 불명확한 건은 자동 반영하지 않고 담당자 검토 대상으로 분리했습니다.

결과: 휴가 문서 조회·전사·대장 갱신 반복을 제거해 담당 시간 99% 감소

Case 04 · Finance

야간근로 수당·택시비 지급 검증 자동화

문제세콤, 근태, 전자결재와 영수증을 사람이 대조해 야근식대·초과수당·택시비를 산정성과재무 반복 정산업무 80% 이상 감소

급여 관련 결과를 LLM이 확정하지 않도록 기본 산정, 택시비 보정, 최종 Validator를 분리했습니다.

1. 3중 교차검증
세콤 퇴근시간, 내부 근태시스템, 전자결재 승인 정보를 대조해 일치하는 근로자의 야간근로 시간과 지급 대상을 산정합니다.
2. Excel 반영
정상 건을 Excel COM 자동화로 지급 대장에 반영하고 원천 근거를 내부 DB에 저장합니다.
3. Taxi Agent 보정
택시비 전자결재를 제출한 임직원을 식별해 영수증과 신청 정보를 기준으로 지급 항목을 보정합니다.
4. 정적 Validator
최종 산출된 야간근로 시간, 전자결재 존재 여부, 영수증 금액과 지급 결과를 결정론적으로 재검증합니다.

통제 포인트

Agent는 문서 해석과 후보 생성에만 사용하고, 최종 금액과 지급 여부는 원천 데이터 기반 Validator가 확인합니다. 불일치 건은 지급 대장에서 분리합니다.

결과: 재무팀의 급여·복리후생 반복 정산 작업을 80% 이상 감소시키고 감사 증적 확보

홈으로 돌아가기 · 회사명, 개인 식별정보, 내부 원문과 실제 데이터는 공개하지 않고 구조와 성과만 익명화했습니다.

Operating Model

업무 자동화를 운영 시스템으로 만드는 방식

1

원천과 승인 흐름 확인

사람이 다시 입력하는 화면보다 이미 승인된 PR, 전자결재, 근태, 출입 기록을 System of Record 후보로 봅니다.

2

정상 건 자동 처리

명확히 일치하는 항목은 자동 반영하고, 충돌하거나 모호한 건만 빨간색 예외 목록으로 올립니다.

3

Agent와 Validator 분리

Agent는 비정형 해석과 후보 생성을 담당하고, 최종 지급·승인·배포는 결정론적 검증기로 통제합니다.

4

증적과 변경 이력 보존

원문 링크, 입력 근거, 수정자, 실행 결과를 DB와 Git에 남겨 재현·감사·롤백이 가능하게 합니다.
